Guide – Plan de Contrôle Cyber

Application de gestion du plan de contrôle cybersécurité. Ce guide couvre l'installation, les fonctionnalités et les procédures d'utilisation.

Installation

Prérequis

  • Python 3.10 ou supérieur
  • Accès Internet pour le premier lancement (CDN Tailwind CSS)

1. Installer les dépendances

Ouvrez un terminal dans le dossier plandecontrole\ :

pip install -r requirements.txt

2. Lancer l'application

python run.py

La base de données, les comptes par défaut et les référentiels sont créés automatiquement au premier lancement.

3. Accéder à l'application

Ouvrez votre navigateur sur http://127.0.0.1:8002

Charger des données de démonstration

python seed_controls.py

Insère 15 contrôles de démonstration répartis sur plusieurs thématiques.

Contrôles

Un contrôle représente une vérification périodique à effectuer dans le cadre du plan de sécurité. Chaque contrôle a une fréquence, un taux cible et peut être assigné à un responsable.

Champs d'un contrôle

RéférenceIdentifiant unique (ex. ACC-001)
LibelléDescription courte du contrôle
IndicateurCe que l'on mesure (ex. nombre de comptes vérifiés)
ObjectifComment vérifier (procédure à suivre)
FréquenceMensuel · Bimestriel · Trimestriel · Semestriel · Annuel
Taux cibleSeuil de conformité en % (défaut : 100%)
ThématiqueDomaine fonctionnel (ex. Accès logique, Alertes…)
CatégorieEntité porteuse (ex. Entité 1, Entité 2)
PérimètreScope organisationnel (ex. SMSI, Tous)
ResponsableUtilisateur responsable du contrôle
Guide / ProcédureLien URL vers le document de procédure externe

Archivage

Un contrôle archivé disparaît de la liste principale et n'est plus suivi. L'archivage est réversible (bouton Restaurer). Tous les résultats et l'historique sont conservés.

Planification annuelle

La planification permet de créer à l'avance toutes les périodes d'un contrôle pour une année donnée, et d'assigner chaque période à un auditeur.

Procédure

  1. Ouvrir la fiche d'un contrôle
  2. Cliquer sur Planifier l'année
  3. Sélectionner l'année à planifier (2024–2050)
  4. Pour chaque période, choisir un auditeur dans le menu déroulant
  5. Cliquer sur Enregistrer la planification

Les périodes déjà clôturées ne peuvent pas être modifiées via la planification.

Si une période existe déjà, la planification met à jour uniquement l'assignation — aucun doublon n'est créé.

Guide / Procédure

Chaque contrôle peut référencer un lien URL vers sa procédure ou son guide opératoire (Confluence, SharePoint, PDF en ligne…).

Ce lien s'affiche dans la fiche du contrôle sous le libellé Guide / Procédure — il s'ouvre dans un nouvel onglet.

Renseignez l'URL complète (avec https://) dans le formulaire de création ou de modification du contrôle.

Campagne mensuelle

La page Campagne affiche la liste de tous les contrôles à réaliser pour un mois donné, quel que soit leur fréquence (un contrôle trimestriel apparaît uniquement lors de son mois de lancement).

Fonctionnalités

  • Navigation mois par mois (← Aujourd'hui →)
  • Indicateurs de progression : total, non saisis, saisis, clôturés
  • Colonne Assigné à : le responsable peut affecter un auditeur directement depuis la liste (la sélection est sauvegardée instantanément)
  • Bouton Saisir pour accéder directement au formulaire de la période

Saisie des résultats

Procédure de saisie

  1. Ouvrir la fiche du contrôle ou passer par la page Campagne
  2. Cliquer sur Saisir (période "À faire") ou Modifier
  3. Saisir le taux de conformité (0–100%)
  4. Le statut est calculé automatiquement : Conforme si taux ≥ taux cible, Non conforme sinon
  5. Ajouter un commentaire si besoin, puis enregistrer

Statuts des résultats

À fairePériode planifiée mais aucun taux saisi
En attenteTaux saisi, en attente de clôture par le responsable
ConformeTaux ≥ taux cible du contrôle
Non conformeTaux < taux cible — action corrective requise
⚠ Incident en coursIncident ouvert suite à une non-conformité — en cours de traitement
N/AContrôle non applicable pour cette période

Gestion des incidents

Lorsqu'un résultat est Non conforme, le responsable peut ouvrir un incident pour suivre la remédiation.

Workflow

Non conforme
├─ [Ouvrir incident] → Incident en cours
└─ [Incident résolu] → Clôturé
└─ [Clôturer quand même] → Clôturé directement

Ouvrir un incident

  1. Sur la fiche du contrôle, cliquer sur ⚠ Ouvrir incident (colonne Clôturé)
  2. Dans la fenêtre modale, saisir un numéro d'incident existant (ex. INC-2026-042) — optionnel
  3. Si JIRA est configuré, cocher Pousser un ticket JIRA pour créer automatiquement un ticket
  4. Cliquer sur Ouvrir l'incident

N° Incident

Une fois l'incident ouvert, le numéro de référence s'affiche dans la colonne N° Incident. Si le numéro n'a pas été saisi lors de l'ouverture, un bouton + Ajouter N° permet de le renseigner ultérieurement.

Un incident peut être lié à un numéro manuel ET à un ticket JIRA simultanément.

Clôtures

La page Clôtures (menu principal) liste tous les résultats saisis mais non encore clôturés par un responsable.

Clôturer depuis la fiche contrôle

Sur la fiche d'un contrôle, dans la colonne Clôturé :

  • Résultat conforme ou N/A → bouton Clôturer
  • Non conforme → bouton ⚠ Ouvrir incident ou Clôturer quand même
  • Incident en cours → bouton ✓ Incident résolu

Rouvrir une clôture

Un résultat clôturé peut être rouvert par un responsable via le bouton Rouvrir. Cela annule la clôture et permet une nouvelle saisie ou révision.

Tableau de bord

Le tableau de bord offre une vue synthétique du plan de contrôle pour une année sélectionnable. Il est organisé en trois onglets — Vue annuelle, Vue mensuelle et Indicateurs — accessibles depuis la barre d'onglets sous les KPIs.

Indicateurs de synthèse (communs aux trois vues)

  • • Total contrôles actifs

  • • Sans résultat (non saisis)
  • • En retard
  • • À clôturer (en attente de validation)

Administration

Accessible depuis le menu Admin (rôle responsable uniquement). Permet de gérer les trois référentiels utilisés dans les contrôles.

RéférentielDescriptionExemples
ThématiquesDomaine fonctionnel du contrôle — avec code couleurAccès logique, Alertes, Audit de config…

La suppression est toujours possible. Si le référentiel est utilisé par des contrôles, ceux-ci seront détachés (le champ correspondant sera vidé). Vous pouvez aussi désactiver un référentiel pour qu'il n'apparaisse plus dans les formulaires sans perdre de données.

Gestion des utilisateurs

Accessible via l'icône Utilisateurs en haut à droite de la navbar (responsable uniquement).

Créer un compte

  1. Cliquer sur Nouveau compte
  2. Renseigner le nom d'utilisateur, le nom complet, l'email et le mot de passe
  3. Choisir le rôle : Auditeur ou Responsable

Modifier un compte

Le mot de passe peut être laissé vide pour le conserver inchangé. Seul le dernier responsable ne peut pas être rétrogradé.

Paramètres

Accessible via l'icône engrenage en haut à droite (responsable uniquement).

Logo société

Déposez un logo (PNG, JPG, SVG, WEBP — max 2 Mo). Il s'affichera dans la navbar.

Thème couleurs

Choisissez parmi 10 palettes de couleurs (Indigo, Blue, Teal, Emerald, Sky, Purple, Rose, Orange, Amber, Slate). La couleur s'applique à la navbar, aux boutons et aux éléments actifs.

Alertes

Configurez les seuils d'alerte :

  • Jours d'avertissement — délai avant la fin de la période où une alerte jaune apparaît
  • Jours de tolérance — délai après la fin de la période avant qu'une alerte rouge (retard) soit déclenchée

LDAP / SSO (Active Directory)

L'application supporte l'authentification via un annuaire Active Directory / LDAP. Une fois activée, les utilisateurs peuvent se connecter avec leurs identifiants de domaine — aucun compte local n'est nécessaire pour eux.

Fonctionnement

  • L'authentification LDAP est tentée en premier. En cas d'échec ou si LDAP est désactivé, le compte local est utilisé en fallback.
  • Lors de la première connexion LDAP, un compte local est créé automatiquement avec le rôle par défaut configuré.
  • À chaque connexion, le nom complet et l'email sont synchronisés depuis l'annuaire.
  • Le mot de passe local n'est pas utilisé pour les comptes authentifiés via LDAP.

Tester la connexion

Le bouton Tester la connexion (en haut à droite du panneau LDAP) vérifie l'accessibilité du serveur sans effectuer d'authentification utilisateur.

⚠ Le compte admin local reste toujours fonctionnel même si LDAP est activé — conservez son mot de passe en lieu sûr.

Les comptes créés automatiquement via LDAP peuvent être retrouvés et modifiés (rôle, désactivation) dans Paramètres → Utilisateurs.

Intégration JIRA

Lorsque JIRA est configuré, l'ouverture d'un incident sur un résultat non conforme peut créer automatiquement un ticket dans votre projet JIRA (API REST v3 — JIRA Cloud).

Configuration (Paramètres → JIRA)

URL JIRAEx. https://monentite.atlassian.net
Email utilisateurCompte de service JIRA
Token APIGénéré depuis votre profil Atlassian (Sécurité → Tokens API)
Clé projetEx. SEC — visible dans les paramètres du projet JIRA
Le ticket créé contient le libellé du contrôle, la période, le taux de conformité et le commentaire. Il est de type Task dans le projet configuré.

Journal d'activité

Accessible depuis le menu Activité. Toutes les actions sont tracées automatiquement : créations, modifications, saisies de résultats, validations, ouvertures d'incidents, connexions.

La colonne Ressource affiche un lien cliquable vers le contrôle concerné. Les entrées sont paginées (50 par page).
